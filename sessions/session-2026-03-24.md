## Topics covered
- Triggers (BEFORE INSERT, UPDATE, DELETE)
- Row-level triggers with FOR EACH ROW
- :OLD and :NEW pseudo-records
- SYSDATE and USER pseudocolumns
- RAISE_APPLICATION_ERROR for custom error handling
- Exception handling with WHEN OTHERS in triggers

## What I understood
- Triggers fire automatically before DML operations, without needing to be called explicitly.
- BEFORE triggers let me intercept and modify :NEW values before they are written to the table — useful for auto-filling audit columns like LOG_DATETIME and CREATED_BY_USER.
- :OLD holds the existing row values before an update or delete, which lets me enforce ownership rules by comparing :OLD.CREATED_BY_USER to the current USER.
- The USER pseudocolumn gives me the currently connected database user, which I can use to restrict who can perform certain operations.
- RAISE_APPLICATION_ERROR lets me return meaningful error codes and messages to the application when a trigger blocks an operation.
- A general WHEN OTHERS exception handler in each trigger ensures no silent failures — any unexpected error gets caught and surfaced cleanly.
