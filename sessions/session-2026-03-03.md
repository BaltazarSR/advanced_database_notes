## What I understood
- Window functions let me perform calculations across rows without collapsing them like GROUP BY does. I can see both individual row data and aggregate calculations at the same time.
- PARTITION BY splits the data into groups, and calculations happen separately within each group. It's like creating mini-datasets within my query.
- The ROWS BETWEEN clause gives me precise control over which rows to include in calculations, which is super useful for moving averages and running totals.
- Sliding windows are essentially a "frame" that moves as the query processes each row, allowing me to calculate things like 7-day moving averages.
