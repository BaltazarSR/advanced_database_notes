# Session – 2026-05-26

## Topics covered
- ETL pipeline: Extract → Transform → Load from an OLTP source into a star schema data warehouse
- OLTP vs Data Warehouse: why transactional tables are bad for analytics and how a star schema fixes that
- Star schema design: fact tables (metrics) and dimension tables (context), surrogate keys
- SCD2 (Slowly Changing Dimension Type 2): storing assignment history with `valid_from` / `valid_to` to track who was responsible at any point in time
- Point-in-time lookup: `valid_from <= ts AND (valid_to IS NULL OR valid_to > ts)` to find the correct record at a given timestamp
- Trigger for assignment logging: AFTER INSERT OR UPDATE fires automatically to close the old assignment row and open a new one
- `MERGE INTO` (upsert): insert if not matched, update if matched — makes the ETL safe to re-run
